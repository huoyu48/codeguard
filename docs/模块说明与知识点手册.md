### 1.5 ✅ `app/models/schemas.py` — 数据模型

**源码**：

```python
"""API 请求/响应模型"""
from pydantic import BaseModel, Field
from datetime import datetime


class WebhookPayload(BaseModel):
    """GitHub Webhook 载荷（简化版）"""
    action: str
    number: int | None = None
    repository: dict = Field(default_factory=dict)
    pull_request: dict | None = None


class ReviewRequest(BaseModel):
    """审查请求"""
    repo_owner: str
    repo_name: str
    pr_number: int
    diff_content: str
    language: str = "python"


class ReviewResult(BaseModel):
    """审查结果"""
    review_id: str
    pr_number: int
    status: str  # "pending" | "completed" | "failed"
    summary: str = ""
    issues_found: int = 0
    created_at: datetime = Field(default_factory=datetime.now)
```

**这段代码做了什么**：定义三个核心数据结构。`WebhookPayload` 是 GitHub 发来的 JSON（只取 4 个关键字段，GitHub 载荷很庞大）。`ReviewRequest` 是内部传递的审查请求，包含 PR 信息和代码 diff。`ReviewResult` 是审查结果，status 有三种状态。注意 `created_at` 用 `default_factory=datetime.now` 而不是 `default=datetime.now()`——后者在类定义时就计算了时间，所有实例会共享同一个时间戳。

**知识点**：

| 知识点 | 说明 | 面试怎么讲 |
|---|---|---|
| **Pydantic BaseModel** | 声明式数据校验 | "FastAPI 自动做参数校验，类型错直接返回 422" |
| **default_factory** | 可变默认值必须用 factory | "datetime.now() 直接写 default 里会所有实例共享同一时间" |
| **int \| None** | Python 3.10+ 联合类型语法 | "PR number 可能为空（push 事件没 PR），用联合类型表示可选" |

### 1.7 📝 `app/middleware/webhook_auth.py` — HMAC 签名校验

**当前状态**：仅有 docstring stub。

**这个模块要做什么**：接收 GitHub Webhook 时，从请求头 `X-Hub-Signature-256` 取出签名，用配置的 `GITHUB_WEBHOOK_SECRET` 对请求 body 重新算 HMAC-SHA256，对比是否一致。不一致返回 401。

**核心逻辑预览**：

```python
import hmac
import hashlib
from fastapi import Request, HTTPException
# ⚠️ 注意：导入的是 get_settings 函数，不是 settings 对象
from app.config import get_settings

settings = get_settings()


async def verify_webhook_signature(request: Request) -> bytes:
    """校验 GitHub Webhook 的 HMAC-SHA256 签名"""
    signature = request.headers.get("X-Hub-Signature-256", "")
    body = await request.body()

    # ⚠️ 注意：config.py 里字段名是 github_webhook_secret，不是 WEBHOOK_SECRET
    expected_signature = "sha256=" + hmac.new(
        settings.github_webhook_secret.encode(),
        body,
        hashlib.sha256,
    ).hexdigest()

    # hmac.compare_digest 防时序攻击，不要用普通 == 比较
    if not hmac.compare_digest(signature, expected_signature):
        raise HTTPException(status_code=401, detail="Invalid signature")

    return body
```

**知识点**：

| 知识点 | 说明 | 面试怎么讲 |
|---|---|---|
| **HMAC-SHA256** | 基于密钥的消息认证码 | "GitHub 用共享密钥对 body 算签名，我们收到后重新计算对比，防伪造" |
| **时序攻击** | 通过比较响应时间逐字节猜签名 | "用 hmac.compare_digest 替代 ==，始终比较完整长度，防时序攻击" |
| **Depends 注入** | 把校验逻辑注入到路由函数 | "签名校验写成 Depends，路由签名加个参数就自动执行" |

### 1.8 📝 `app/api/webhook.py` — Webhook 接收端点

**当前状态**：空目录，待实现。

**这个模块要做什么**：定义 `POST /api/webhook` 路由。收到 GitHub 推送后：校验签名 → 过滤事件（只处理 pull_request 的 opened / synchronize） → 提取信息 → 入队 Celery → 返回 202 Accepted。

**核心逻辑预览**：

```python
from fastapi import APIRouter, Depends, HTTPException
# ⚠️ 前提：先完成 1.7 webhook_auth.py，否则 import 会报错
from app.middleware.webhook_auth import verify_webhook_signature
from app.models.schemas import WebhookPayload
# ⚠️ 前提：先完成 1.9 review_task.py，否则 import 会报错
from app.tasks.review_task import process_review

router = APIRouter()


@router.post("/api/webhook", status_code=202)
async def receive_webhook(
    body: bytes = Depends(verify_webhook_signature),
):
    """接收 GitHub Webhook 事件"""
    import json  # json 是标准库，放函数里也没问题，只是习惯写法
    payload = WebhookPayload(**json.loads(body))

    # 只处理 PR 的 opened 和 synchronize 事件
    if payload.action not in ("opened", "synchronize"):
        return {"message": f"Ignored action: {payload.action}"}

    if not payload.pull_request:
        raise HTTPException(400, "No pull_request in payload")

    # 异步入队
    process_review.delay(
        repo_owner=payload.repository["owner"]["login"],
        repo_name=payload.repository["name"],
        pr_number=payload.number,
    )

    return {"message": "Review task queued", "pr": payload.number}
```

**知识点**：

| 知识点 | 说明 | 面试怎么讲 |
|---|---|---|
| **HTTP 202 vs 200** | 202 = 已接受但未处理完 | "审查是异步的，202 语义比 200 更准确" |
| **事件过滤** | GitHub 发很多种事件 | "只处理 opened 和 synchronize，其他直接忽略" |
| **Depends 注入** | 签名校验作为依赖注入 | "每个请求自动过签名校验，不需要手动调" |

```python

from app.config import get_settings

settings = get_settings()

celery_app = Celery(
    "codeguard",
    broker=settings.redis_url,
    backend=settings.redis_url,

celery_app.conf.update(
    task_serializer="json",
    accept_content=["json"],
    timezone="Asia/Shanghai",

    worker_prefetch_multiplier=1, # 每次只取一个任务，避免 Worker 卡死
)
```

```python
# review_task.py
from app.tasks.celery_app import celery_app


@celery_app.task(
    bind=True,
    max_retries=3,
    default_retry_delay=10,
    name="process_review",
)
def process_review(self, repo_owner: str, repo_name: str, pr_number: int):
    """异步执行代码审查"""
    try:
        # 1. 获取 PR diff
        # 2. 调用 Agent Pipeline
        # 3. 回写审查评论到 GitHub
        pass
    except Exception as exc:
        # 指数退避重试：10s → 20s → 40s
        raise self.retry(
            exc=exc,
            countdown=self.default_retry_delay * (2 ** self.request.retries),

```

**知识点**：

| 知识点 | 说明 | 面试怎么讲 |
|---|---|---|
| **Celery** | 分布式任务队列 | "Webhook 入队 < 100ms，Worker 异步消费，解耦接收和处理" |
| **bind=True** | 任务函数能访问 self | "通过 self.retry() 重试，self.request.retries 获取重试次数" |
| **task_acks_late** | 完成后才确认 | "Worker 崩溃时任务自动重入队，不会丢任务" |
| **水平扩展** | 加 Worker 就行 | "100 个 PR 同时来，开多个 Worker 并行消费" |
| **asyncio vs Celery** | 协程 vs 进程级分布式 | "asyncio 适合单进程 I/O 密集，Celery 支持多进程扩展和重试" |

```python
import httpx
from app.config import get_settings

settings = get_settings()


class GitHubService:
    BASE_URL = "https://api.github.com"

    def __init__(self, access_token: str):
        self.client = httpx.AsyncClient(
            headers={
                "Authorization": f"token {access_token}",
                "Accept": "application/vnd.github.v3+json",
            },
            timeout=30,
        )

    async def get_pr_diff(self, owner: str, repo: str, pr_number: int) -> str:
        resp = await self.client.get(
            f"{self.BASE_URL}/repos/{owner}/{repo}/pulls/{pr_number}",
            headers={"Accept": "application/vnd.github.v3.diff"},
        )
        resp.raise_for_status()
        return resp.text

    async def post_review_comment(
        self, owner: str, repo: str, pr_number: int, body: str
    ) -> dict:
        resp = await self.client.post(
            f"{self.BASE_URL}/repos/{owner}/{repo}/issues/{pr_number}/comments",
            json={"body": body},
        )
        resp.raise_for_status()
        return resp.json()
```

### 2.1 📝 `app/agent/tools.py` — Agent 工具集

**当前状态**：空目录，待实现。

**这个模块要做什么**：用 LangChain `@tool` 装饰器定义三个工具：`analyze_code_structure`（AST 解析 Python 代码的类、函数、导入）、`scan_security_issues`（正则匹配 eval()、硬编码密钥等不安全模式）、`calculate_complexity`（统计 if/for/while/except 分支数计算圈复杂度）。Agent 在审查中自主选择调用哪些。

**核心逻辑预览**：

```python
import ast
import re
from langchain_core.tools import tool


@tool
def analyze_code_structure(code: str) -> str:
    """分析 Python 代码结构，提取类、函数、导入信息。
    当需要了解代码的整体组织和模块划分时使用。"""
    tree = ast.parse(code)
    classes = [n.name for n in ast.walk(tree) if isinstance(n, ast.ClassDef)]
    functions = [n.name for n in ast.walk(tree) if isinstance(n, ast.FunctionDef)]
    imports = [
        n.names[0].name
        for n in ast.walk(tree)
        if isinstance(n, ast.Import)
    ]
    return f"Classes: {classes}\nFunctions: {functions}\nImports: {imports}"


@tool
def scan_security_issues(code: str) -> str:
    """扫描代码中的安全隐患（eval/exec 调用、硬编码密钥、SQL 拼接等）。
    当需要检查代码安全性时使用。"""
    patterns = {
        "eval() 调用": r"\beval\s*\(",
        "exec() 调用": r"\bexec\s*\(",
        "硬编码密钥": r'(password|secret|api_key)\s*=\s*["\']',
        "SQL 拼接": r'(execute|cursor)\s*\(.*%s|f".*SELECT',
    }
    issues = []
    for name, pattern in patterns.items():
        if re.search(pattern, code):
            issues.append(f"[安全] 发现 {name}")
    return "\n".join(issues) if issues else "未发现安全隐患"


@tool
def calculate_complexity(code: str) -> str:
    """计算代码的圈复杂度（分支数量）。
    当需要评估代码是否需要重构时使用。圈复杂度 > 10 建议重构。"""
    tree = ast.parse(code)
    complexity = 1
    for node in ast.walk(tree):
        if isinstance(node, (ast.If, ast.For, ast.While, ast.ExceptHandler)):
            complexity += 1
    level = "低" if complexity <= 5 else "中" if complexity <= 10 else "高（建议重构）"
    return f"圈复杂度: {complexity} ({level})"
```

| 知识点 | 说明 | 面试怎么讲 |
|---|---|---|
| **@tool 装饰器** | LangChain 工具定义方式 | "函数名 = 工具名，docstring = 描述，LLM 根据描述决定何时调用" |
| **AST 解析** | 抽象语法树分析代码结构 | "用 ast 模块提取类和函数，全程只读不执行" |
| **圈复杂度** | 衡量代码复杂程度 | "统计分支数，超过 10 建议重构" |

### 2.2 📝 `app/agent/pipeline.py` — LangGraph Agent 流水线

**当前状态**：空目录，待实现。

**这个模块要做什么**：项目的核心。用 LangGraph 的 StateGraph 构建 Agent 推理流程：reviewer 节点调 LLM 思考 → 检查是否需要工具 → tools 节点执行工具 → 观察结果 → 继续思考或输出结论。后续（第 3 周）会扩展为多 Agent 并行。

**核心逻辑预览**：

```python
from typing import TypedDict
from langchain_openai import ChatOpenAI
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode
# ⚠️ 前提：先完成 2.1 tools.py，否则这里 import 会报错
from app.agent.tools import (
    analyze_code_structure,
    scan_security_issues,
    calculate_complexity,
)
from app.config import get_settings

settings = get_settings()

tools = [analyze_code_structure, scan_security_issues, calculate_complexity]

# ⚠️ 虽然参数名带 openai_，但 DeepSeek 兼容 OpenAI 接口，这里传的是 DeepSeek 的 key 和 url
llm = ChatOpenAI(
    model=settings.deepseek_model,
    temperature=0.2,           # 代码审查需要确定性，不需要创造性
    openai_api_key=settings.deepseek_api_key,      # DeepSeek API Key
    openai_api_base=settings.deepseek_base_url,    # https://api.deepseek.com
).bind_tools(tools)


class AgentState(TypedDict):
    messages: list


def reviewer(state: AgentState):
    """Agent 推理节点：思考下一步该做什么"""
    response = llm.invoke(state["messages"])
    return {"messages": state["messages"] + [response]}


def should_continue(state: AgentState):
    """条件边：Agent 是否还需要调用工具"""
    last_message = state["messages"][-1]
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:
        return "tools"
    return END


def build_review_pipeline():
    """构建 Agent 审查流水线"""
    graph = StateGraph(AgentState)
    graph.add_node("reviewer", reviewer)
    graph.add_node("tools", ToolNode(tools))
    graph.set_entry_point("reviewer")
    graph.add_conditional_edges(
        "reviewer", should_continue, {"tools": "tools", END: END}
    )
    graph.add_edge("tools", "reviewer")  # 工具执行完回到 reviewer 继续思考
    return graph.compile()
```

**知识点**：

| 知识点 | 说明 | 面试怎么讲 |
|---|---|---|
| **ReAct 模式** | Reasoning + Acting 交替 | "思考 → 调工具 → 观察 → 继续思考，循环直到出结论" |
| **StateGraph** | LangGraph 状态图编排 | "节点是执行单元，条件边动态路由下一个节点" |
| **条件边** | 根据状态选择下一步 | "检查最后一条消息有没有 tool_calls，有就执行工具，没有就结束" |
| **temperature=0.2** | 低温度 = 高确定性 | "代码审查要一致性，不像聊天要创造性" |

import chromadb
from chromadb.utils import embedding_functions
from langchain.text_splitter import RecursiveCharacterTextSplitter


class KnowledgeBase:
    def __init__(self, persist_dir: str = "./chroma_data"):
        self.client = chromadb.PersistentClient(path=persist_dir)
        self.embedder = embedding_functions.DefaultEmbeddingFunction()
        self.collection = self.client.get_or_create_collection(
            name="coding_standards",
            embedding_function=self.embedder,
        )

    def load_documents(self, file_path: str):
        """加载编码规范文档到知识库"""
        with open(file_path, "r") as f:
            text = f.read()
        splitter = RecursiveCharacterTextSplitter(
            chunk_size=500, chunk_overlap=50
        )
        chunks = splitter.split_text(text)
        self.collection.add(
            documents=chunks,
            ids=[f"doc_{i}" for i in range(len(chunks))],
        )

    def search(self, query: str, top_k: int = 5) -> list[str]:
        """检索与代码片段最相关的规范条目"""
        results = self.collection.query(
            query_texts=[query], n_results=top_k
        )
        return results["documents"][0]

### 4.1 📝 `app/middleware/rate_limiter.py` — 滑动窗口限流器

**当前状态**：空目录，待实现。

**这个模块要做什么**：限制每个来源在时间窗口内的请求数。用 Redis Sorted Set 实现——每个请求以时间戳为 score 存入，每次先清窗口外过期记录，再统计窗口内请求数，超阈值返回 429。

**核心逻辑预览**：

```python
import time
import redis.asyncio as redis
from fastapi import Request, HTTPException


class SlidingWindowRateLimiter:
    def __init__(
        self,
        redis_client: redis.Redis,
        max_requests: int = 60,
        window_seconds: int = 60,
    ):
        self.redis = redis_client
        self.max_requests = max_requests
        self.window = window_seconds

    async def is_allowed(self, key: str) -> bool:
        now = time.time()
        window_start = now - self.window
        pipe = self.redis.pipeline()
        pipe.zremrangebyscore(key, 0, window_start)  # 清除窗口外的记录
        pipe.zadd(key, {str(now): now})               # 添加当前请求
        pipe.zcard(key)                                # 统计窗口内请求数
        pipe.expire(key, self.window)                  # 设置 key 过期
        results = await pipe.execute()
        return results[2] <= self.max_requests
```

### 4.2 📝 `app/middleware/circuit_breaker.py` — 熔断器

**当前状态**：空目录，待实现。

**这个模块要做什么**：当 DeepSeek API 或 GitHub API 连续失败 N 次后，自动"熔断"——直接拒绝请求而不是继续超时。冷却期后进入"半开"状态试探一个请求，成功则恢复。

**核心逻辑预览**：

```python
import time
from enum import Enum


class CircuitState(Enum):
    CLOSED = "closed"       # 正常放行
    OPEN = "open"           # 熔断，拒绝请求
    HALF_OPEN = "half_open" # 试探性放行


class CircuitBreaker:
    def __init__(
        self, failure_threshold: int = 5, recovery_timeout: int = 30
    ):
        self.failure_threshold = failure_threshold
        self.recovery_timeout = recovery_timeout
        self.failure_count = 0
        self.state = CircuitState.CLOSED
        self.last_failure_time = 0

    def call(self, func, *args, **kwargs):
        if self.state == CircuitState.OPEN:
            if time.time() - self.last_failure_time > self.recovery_timeout:
                self.state = CircuitState.HALF_OPEN  # 冷却期过，试探恢复
            else:
                raise Exception("Circuit breaker is OPEN")

        try:
            result = func(*args, **kwargs)
            self._on_success()
            return result
        except Exception as e:
            self._on_failure()
            raise

    def _on_success(self):
        self.failure_count = 0
        self.state = CircuitState.CLOSED

    def _on_failure(self):
        self.failure_count += 1
        self.last_failure_time = time.time()
        if self.failure_count >= self.failure_threshold:
            self.state = CircuitState.OPEN
```

**知识点**：

| 知识点 | 说明 | 面试怎么讲 |
|---|---|---|
| **三态流转** | Closed → Open → Half-Open | "连续失败 N 次熔断，冷却后试探一个请求" |
| **雪崩效应** | 下游挂了重试堆积拖垮上游 | "不做熔断，线程池耗尽后整个系统崩" |
| **指数退避** | 重试间隔指数增长 | "10s → 20s → 40s，不给过载的下游更大压力" |
